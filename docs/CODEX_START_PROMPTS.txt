项目最终定位：
AsyncDataLoader 是一个面向科学计算离线数据加工场景的大规模数据异步预处理流水线。
核心流程是：

read raw block -> CPU preprocessing stage -> write processed block

这个项目不是普通文件加载器。
它最终必须体现：

1. 有界内存；
2. 背压；
3. 流式处理；
4. read/process/write 三级重叠；
5. 真实 CPU processing stage；
6. 清晰 buffer 所有权；
7. RAII 归还 buffer；
8. 输出顺序正确；
9. 可靠落盘；
10. backend fallback；
11. metrics；
12. benchmark；
13. 错误测试；
14. 面试可解释的工程边界。

防坍缩硬约束：
任何时候都不能为了省事破坏下面约束：

1. 不允许把整个输入文件读入内存。
2. 不允许把所有 block 攒进 vector 后再统一处理。
3. 不允许使用无界队列作为主路径。
4. 不允许让 reader 无限超前于 processor/writer。
5. 串行 read-process-write 只能作为 baseline，不能作为最终 pipeline。
6. NoOp 或 checksum-only 可以作为早期 demo，但不能作为最终 processing stage 的证明。
7. 不允许删掉 BufferPool、backpressure、metrics 或可靠落盘机制来简化代码。
8. 不允许编造 benchmark 数字。
9. 不允许声称“协程本身带来性能提升”。
10. 不允许声称“io_uring 永远最快”。
11. 不允许擅自引入 CUDA、AI Infra、分布式系统、Dashboard、数据库、生产部署或具体学科文件格式解析。
